#### Установка и настройка утилит SmartView
## Установка утилит SmartView

Для начала работы необходимо установить утилиты SmartView.

### Шаги по установке:
1. Установить Git через SUS.
2. Добавить настройку конфигурации `git config --global http.sslVerify false` единожды.
3. Клонировать репозиторий утилит SmartView: `git clone https://example.com/smartview/tools-repo.git`.
4. Установить необходимые зависимости: `requests`, `requests_negotiate_sspi`, `pandas`.

### Инструкции по работе с утилитами:
#### Сбор информации о камерах
- Запустить Jupyter тетрадку `get_info_table` и выполнить все ячейки.
#### Массовое получение снимков
- Создать список идентификаторов камер (`cams_ids_list`) и выполнить цикл для получения снимков.
#### Заказ видео из архива
- Определить интервал времени и вызвать функцию `order_video` для конкретного ID камеры.

#### Создание отчета на QS-дэшборде
### Отчет на QS-дэшборде
Для формирования отчета используются утилиты SmartView.

#### Процесс формирования отчета:
1. Собрать данные с помощью утилит SmartView.
2. Отправить данные на сервер через API.
3. Проанализировать и визуализировать данные на дэшборде QS.

#### Описание ограничений:
Отчет доступен только после успешной интеграции утилит SmartView и сервера QS.

#### Передача аргументов функции order_video()
Функция `order_video(session, camId , period)` принимает следующие аргументы:
- session: сессия аутентификации
- camId: идентификатор камеры
- period: временной период, представленный двумя датами формата 'YYYY-MM-DD HH:MM:SS'
Пример корректного вызова функции: `order_video(get_session(), 'CAMERA_ID', ('2025-01-28 11:00:00', '2025-01-28 12:15:00'))`.

#### Обработка результатов Jupyter-тетрадок
После выполнения Jupyter-тетрадок необходимо обработать и интерпретировать полученные результаты следующим образом:
- Провести анализ полученных данных.
- Выбрать подходящие методы анализа в зависимости от типа данных.
- Представить результаты в удобочитаемом виде, используя визуализации и отчеты.
